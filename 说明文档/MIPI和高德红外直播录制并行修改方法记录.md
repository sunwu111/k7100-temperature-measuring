# MIPI和高德红外直播录制并行修改方法记录

## 一、背景

本次记录的是两类设备在“直播拉流”和“录制短视频”同时进行时的修改方法：

+ MIPI摄像头
+ 高德红外机芯，也就是 `GUIDEDev`

目标不是优化帧率，也不是改算法处理链路，而是解决业务互斥问题：

+ 点击录制短视频时，不能打断正在进行的直播。
+ 停止直播时，不能影响正在录制的短视频。
+ 录制短视频结束时，不能影响仍在进行的直播。
+ 两个业务共用底层帧源时，只释放当前业务自己的资源。

## 二、MIPI摄像头修改方法

### 1. 调度入口处理

MIPI点击录制短视频时，上层入口不能再因为设备正在直播就主动停止直播。

处理方法：

+ 在手动录制短视频入口中，MIPI正在直播时只记录日志，不调用停止直播。
+ 后续仍然进入原有打开设备和录制流程。
+ 如果设备已经在直播中，打开设备时直接复用当前Camera2会话，不重新打开摄像头。

这样可以避免“点击录制短视频的一瞬间，直播先被上层停掉”。

### 2. Camera2会话复用

MIPI直播和短视频录制使用同一个Camera2帧源。

处理方法：

+ 如果直播或录制已经在运行，并且当前相机会话可用，则 `open` 流程直接返回成功。
+ 不重新创建Camera2设备。
+ 不重新创建预览会话。
+ 不重新切换MIPI硬件。

这样可以避免录制启动时重新打开相机导致直播帧源中断。

### 3. 编码器复用

MIPI直播和短视频录制共用同一个视频编码器。

处理方法：

+ 直播中启动录制时，复用当前编码器。
+ 录制中启动直播时，只创建直播侧RTP封包器。
+ 编码后的同一帧同时分发给两个消费者：录像写MP4，直播发RTP。
+ 停止直播时，如果录制仍在运行，不释放编码器。
+ 停止录制时，如果直播仍在运行，不释放编码器。

这样可以避免“录制启动时重建编码器，直播编码链路被打断”。

### 4. 时间戳隔离

录制写MP4时会调整录像时间戳，直播RTP也需要使用编码帧时间戳。

处理方法：

+ 录像写入使用单独复制出来的时间戳信息。
+ 直播RTP继续使用编码器原始输出时间戳。
+ 录像写入异常只记录日志，不阻断直播RTP发送。

这样可以避免录制过程中修改时间戳，导致直播RTP时间戳跳变。

### 5. RTP状态重置

如果录制过程中停止过直播，再重新开始直播，需要重新初始化直播RTP状态。

处理方法：

+ 创建新的RTP封包器后，重置首帧时间戳。
+ 清空旧的SPS/PPS缓存。

这样可以避免新直播会话沿用旧会话状态。

## 三、高德红外修改方法

### 1. 高德红外机芯连接复用

高德红外点击录制短视频时，也会走设备打开流程。如果此时重新打开USB红外机芯，直播会立即断开。

处理方法：

+ 如果高德红外正在直播或正在录制，`open` 流程直接复用当前机芯连接。
+ 不重新打开USB机芯。
+ 不重新注册预览源。

这样可以避免点击录制时因为重新打开红外机芯导致直播断流。

### 2. 预览线程复用

高德红外直播和录制都来自同一个USB红外预览帧源。

处理方法：

+ 直播中启动录制时，不再启动第二条预览线程。
+ 录制中启动直播时，不再启动第二条预览线程。
+ 当前预览线程的循环条件改为：直播或录制任一业务仍在运行，就继续取帧。
+ 只有直播和录制都结束后，才允许停止预览。

这样可以避免一个业务停止时，把另一个业务仍在使用的红外预览源关闭。

### 3. 视频编码器复用

高德红外当前也是把预览Bitmap送入父类编码流程。

处理方法：

+ 直播中启动录制时，复用当前视频编码器。
+ 录制中启动直播时，只创建直播侧RTP封包器。
+ 停止录制时，如果直播还在，保留视频编码器。
+ 停止直播时，如果录制还在，保留视频编码器。

这样可以避免直播和录制互相重建编码器。

### 4. 业务输出分离

高德红外虽然共用预览和编码器，但直播和录制的输出目标不同。

处理方法：

+ 录制负责打开Muxer并写MP4。
+ 直播负责创建RTP封包器并发送直播帧。
+ 停止直播只清理直播侧RTP状态。
+ 停止录制只清理录像侧Muxer状态。

这样可以避免一个业务清理资源时误清理另一个业务。

## 四、共同原则

两类设备的共同修改原则如下：

+ 帧源共享：同一个设备同一时刻只保留一个底层帧源。
+ 编码共享：在当前实现下，直播和录制共用一个编码器。
+ 输出分离：直播输出走RTP，录像输出走MP4。
+ 状态分离：直播状态和录制状态分别维护。
+ 资源释放延后：只有直播和录制都结束后，才释放底层帧源和编码器。
+ 启动复用优先：业务运行中再次打开设备时，优先复用当前连接，不重新打开设备。

## 五、验证方法

建议按以下顺序验证：

+ 先启动直播，再点击录制短视频，确认直播不中断，录像文件正常生成。
+ 录制过程中停止直播，确认短视频继续录制到结束。
+ 直播过程中等待短视频录制结束，确认直播仍然保持。
+ 先启动录制短视频，再启动直播，确认直播可以出流，录像继续写入。
+ 两个业务都结束后，再确认设备资源可以正常释放。

关键日志可以关注：

+ MIPI录制短视频时保留当前直播
+ MIPI摄像头业务运行中，复用当前相机会话
+ MIPI直播中启动录制，复用当前预览会话和视频编码器
+ MIPI录制中启动拉流成功，共用当前预览会话和视频编码器
+ 高德红外业务运行中，复用当前机芯连接
+ 高德红外直播中启动录制，复用当前预览和视频编码器
+ 高德红外录制中启动直播，复用当前预览和视频编码器
+ 高德红外录制停止，直播仍在进行，保留视频编码器
