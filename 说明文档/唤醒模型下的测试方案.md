# 唤醒模型下的测试方案

## 1. 测试目标

验证设备处于唤醒模式时，通道一和通道二的业务任务是否能正确唤醒设备、执行任务、进入保活，并在保活期内和保活期后表现符合预期。

重点验证：

- 任务首次执行时是否会唤醒云台。
- 保活期内再次执行任务是否正常。
- 保活结束后再次执行任务是否能重新唤醒，并重新等待设备就绪。
- 定时拍照是否会提前开启云台。
- 删除或修改拍照策略后，提前唤醒闹钟是否同步取消或重建。
- 任务结束后是否按规则关闭云台和红外。

## 2. 前置条件

- 设备已安装当前 release 版本。
- 设备当前处于唤醒模式。
- 通道一为可见光通道。
- 通道二为可见光或球机通道，按现场实际配置验证。
- 云台、红外初始状态建议先确认处于下电状态。
- 使用设备日志观察：

```powershell
adb shell tail -f /sdcard/zhjinrui/spgp/logs.log
```

关键日志关键词：

- `切换到唤醒模式`
- `开启云台`
- `云台上电成功`
- `isDeviceReady()：false`
- `isDeviceReady()：true`
- `打开球机耗时`
- `登录成功`
- `直播预览：成功`
- `抓拍成功`
- `录制成功`
- `回放`
- `设备处于唤醒模式，并在保活任务后的2分钟内，不进行关闭操作`
- `关闭云台`
- `云台下电成功`

## 3. 通用判定标准

首次开启任务：

- 设备应打印 `开启云台`。
- 云台应上电成功。
- 如果设备刚下电后重新上电，应看到 `isDeviceReady()：false` 到 `isDeviceReady()：true` 的等待过程。
- 任务应执行成功。
- 任务结束后应进入 2 分钟保活，不应立即下电。

保活期内再次开启任务：

- 任务应能正常执行。
- 不应因为上一次任务残留状态导致失败。
- 任务结束后应刷新保活时间。
- 在保活期内不应关闭云台。

保活后再次开启任务：

- 保活到期后应看到云台下电日志。
- 再次执行任务时，应重新开启云台。
- 重新上电后应重新等待设备就绪，不能直接复用旧 `onvifReady/session`。
- 任务应执行成功。

异常判定：

- `设备POST失败`
- `SocketTimeoutException`
- `login::response is null`
- `HTTP请求失败`
- `ONVIF直播失败`
- `直播预览：失败`
- `抓拍失败`
- `录制失败`
- 保活期内提前下电
- 保活后再次执行任务没有重新等待设备就绪

## 4. 通道一测试项

### 4.1 通道一拉流

步骤：

1. 确认设备处于唤醒模式，云台已下电。
2. 对通道一发起拉流。
3. 观察首次拉流日志。
4. 停止拉流。
5. 在 2 分钟保活期内再次发起通道一拉流。
6. 停止拉流，等待保活超过 2 分钟。
7. 再次发起通道一拉流。

预期：

- 首次拉流应开启云台并成功拉流。
- 刚上电时应等待 `isDeviceReady() false -> true`。
- 停止拉流后进入保活，不立即下电。
- 保活期内再次拉流成功。
- 保活后云台下电，再次拉流应重新上电并重新等待设备就绪。

### 4.2 通道一手动拍照

步骤：

1. 确认云台已下电。
2. 对通道一执行手动拍照。
3. 观察拍照、上传、保活日志。
4. 保活期内再次执行手动拍照。
5. 等待保活结束后再次执行手动拍照。

预期：

- 首次手动拍照会开启云台。
- 拍照成功并生成图片。
- 图片上传成功。
- 保活期内再次拍照成功，不应下电。
- 保活后再次拍照应重新上电并等待设备就绪。

### 4.3 通道一定时拍照和提前开启云台

步骤：

1. 设置通道一定时拍照时间为当前时间 5 分钟后。
2. 观察是否生成提前 3 分钟开启云台闹钟。
3. 到达拍照前 3 分钟时，观察云台是否提前上电。
4. 到达拍照时间时，观察拍照是否成功。
5. 修改或删除该拍照策略。
6. 观察旧的提前开启云台闹钟是否被取消。
7. 再设置新的定时拍照策略，确认只存在新的提前开启闹钟。

预期：

- 设置策略后应看到类似 `云台抓拍前3分钟开启` 的闹钟日志。
- 拍照前 3 分钟应触发 `POWERON_RGB_INTENT` 并开启云台。
- 拍照时间到达时应直接执行拍照，不应因为云台未上电失败。
- 删除策略后，原提前开启云台闹钟不应再触发。
- 修改策略后，旧闹钟应取消，新闹钟应按新时间生成。

### 4.4 通道一录制短视频

步骤：

1. 确认云台已下电。
2. 对通道一发起短视频录制。
3. 观察录制开始、录制结束、上传和保活日志。
4. 保活期内再次录制短视频。
5. 保活后再次录制短视频。

预期：

- 首次录制应开启云台。
- 录制期间设备处于忙状态，不应下电。
- 录制完成后进入保活。
- 保活期内再次录制成功。
- 保活后再次录制应重新上电并等待设备就绪。

### 4.5 通道一播放录像回放

步骤：

1. 确认云台已下电。
2. 查询通道一录像文件。
3. 对通道一发起录像回放。
4. 停止回放。
5. 保活期内再次回放。
6. 保活后再次回放。

预期：

- 回放会唤醒云台。
- 回放开始成功，停止回放成功。
- 回放期间设备处于忙状态，不应下电。
- 停止回放后进入保活。
- 保活后再次回放应重新上电并等待设备就绪。

## 5. 通道二测试项

步骤：

1. 确认云台已下电。
2. 对通道二发起拉流。
3. 停止拉流。
4. 保活期内再次拉流。
5. 保活后再次拉流。

预期：

- 首次拉流应开启云台。
- 拉流成功。
- 停止后进入保活。
- 保活期内再次拉流成功。
- 保活后再次拉流应重新上电并等待设备就绪。

### 5.2 通道二手动拍照

步骤：

1. 确认云台已下电。
2. 对通道二执行手动拍照。
3. 保活期内再次手动拍照。
4. 保活后再次手动拍照。

预期：

- 首次拍照应开启云台。
- 拍照成功并上传。
- 保活期内再次拍照成功。
- 保活后再次拍照应重新上电并等待设备就绪。

### 5.3 通道二录制短视频

步骤：

1. 确认云台已下电。
2. 对通道二录制短视频。
3. 等待录制结束。
4. 保活期内再次录制。
5. 保活后再次录制。

预期：

- 首次录制应开启云台。
- 录制期间设备忙，不应下电。
- 录制完成后进入保活。
- 保活期内再次录制成功。
- 保活后再次录制应重新上电并等待设备就绪。

### 5.4 通道二提前开启云台

步骤：

1. 设置通道二定时拍照时间为当前时间 5 分钟后。
2. 观察是否生成提前 3 分钟开启云台闹钟。
3. 到达提前 3 分钟时间点，观察云台是否开启。
4. 删除或修改通道二拍照策略。
5. 观察旧的提前开启云台闹钟是否被取消。

预期：

- 通道二如果是可见光 DVR 通道，应生成提前 3 分钟开启云台闹钟。
- 删除策略后，提前开启云台闹钟不应继续触发。
- 修改策略后，旧闹钟应取消，新闹钟应按新时间触发。

## 6. 保活专项测试

### 6.1 保活期内连续任务

步骤：

1. 通道一手动拍照。
2. 任务结束后 30 秒内通道一拉流。
3. 停止拉流后 30 秒内通道一录制短视频。
4. 录制结束后 30 秒内通道二手动拍照。

预期：

- 每个任务都成功。
- 任务之间不应出现云台下电。
- 每次任务结束后都应刷新保活时间。
- 最后一个任务结束后，等待 2 分钟才允许下电。

### 6.2 保活结束后重新任务

步骤：

1. 执行任意任务并等待成功结束。
2. 不再执行任何操作，等待超过 2 分钟。
3. 确认云台下电。
4. 再执行同类型任务。

预期：

- 保活到期后云台下电。
- 再次执行任务时重新上电。
- 应重新等待设备就绪。
- 不应出现 `login::response is null` 或 HTTP 连接超时。

### 6.3 忙状态保护

步骤：

1. 开始短视频录制。
2. 录制尚未结束时等待保活下电时间点。
3. 观察是否下电。

预期：

- 录制期间不应下电。
- 日志应体现设备忙或延迟关闭。
- 录制完成后才进入保活倒计时。

## 7. 拍照策略删除和修改专项测试

### 7.1 删除单个通道拍照策略

步骤：

1. 设置通道一定时拍照，并确认生成提前 3 分钟闹钟。
2. 删除通道一拍照策略。
3. 等待原提前唤醒时间点。

预期：

- 原提前唤醒闹钟不触发。
- 不应出现 `抓拍前三分钟开启云台`。
- 云台不应因为已删除策略而上电。

### 7.2 修改拍照时间

步骤：

1. 设置通道一定时拍照为时间 A。
2. 确认生成时间 A 对应提前闹钟。
3. 修改为时间 B。
4. 等待时间 A 的提前唤醒点。
5. 等待时间 B 的提前唤醒点。

预期：

- 时间 A 的提前唤醒不再触发。
- 时间 B 的提前唤醒正常触发。
- 最终只执行新策略对应的拍照。

### 7.3 多通道策略互不影响

步骤：

1. 设置通道一和通道二各一个定时拍照策略。
2. 删除通道一策略。
3. 保留通道二策略。
4. 分别观察两个通道原定时间点。

预期：

- 通道一的提前唤醒和拍照不再触发。
- 通道二的提前唤醒和拍照仍正常触发。

## 8. 可能需要重点关注的问题

- 设置拍照策略后没有生成提前唤醒闹钟。
- 删除拍照策略后，旧的提前唤醒闹钟仍触发。
- 保活期内任务结束后，云台提前下电。
- 保活后再次执行任务，没有重新等待 `isDeviceReady()`。
- 下电后很快重新拉流，出现 HTTP 超时或 `login::response is null`。
- 多个同一时间点拍照任务只执行了一个。
- 通道一和通道二的闹钟 requestCode 冲突。
- 红外通道提前 10 分钟唤醒误触发到可见光通道。
- 可见光通道提前 3 分钟唤醒误触发到红外通道。
- 定时拍照策略跨天时，提前唤醒时间是否正确。
- 当前时间已经进入提前唤醒窗口时，是否会立即触发预热。
