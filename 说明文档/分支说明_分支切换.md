### 分支切换


切换提交：git switch --detach 十六进制位数

切换到寰宇直播与录像 RTSP 拆分分支：

```bash
git switch feature/huanyu-rtsp-split
```

低照度参数输入
git switch feature/low-light-gamma-param


处理 MIPI 和红外出流比较慢的问题
git switch feature/mipi-ir-stream-startup

升级日志显示优化
upgrade-log-throttle-20260612-1528


切换到主开发分支：

```bash
git switch test-development
```

`DV分支是test-development的旧分支，江门出差后修改了很多，就重新上传了一个新的分支`


### 合并 feature/huanyu-rtsp-split 到 test-development

合并前先确认当前没有未提交改动：

```bash
git status
```

切换到主开发分支，并拉取远端最新代码：

```bash
git switch test-development
git pull origin test-development
```

合并功能分支：

```bash
git merge feature/huanyu-rtsp-split
```

如果没有冲突，执行编译验证：

```bash
.\gradlew.bat :app:compileDebugJavaWithJavac
```

验证通过后推送到远端：

```bash
git push origin test-development
```

### 合并时出现冲突

查看冲突文件：

```bash
git status
```

手动处理冲突后，标记已解决并提交：

```bash
git add 冲突文件路径
git commit
```

然后重新编译验证，并推送：

```bash
.\gradlew.bat :app:compileDebugJavaWithJavac
git push origin test-development
```

### 不想继续合并

如果合并过程中发现不应该合并，可以中止本次合并：

```bash
git merge --abort
```
