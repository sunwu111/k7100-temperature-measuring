# 电源管理设计文档

## 1. 设计目标

电源管理用于根据设备电量状态，在保证业务任务正常执行的前提下，控制云台、红外、USB、网络等负载的上下电，降低空闲功耗。

系统划分为三个电源模式：

| 模式 | 电量状态 | 主要策略 |
| --- | --- | --- |
| 全工作模式 | 电量充足 | 设备按工作时间保持业务能力。 |
| 基础模式 | 电量中等 | 默认关闭高功耗负载，只有业务需要时临时唤醒。 |
| 休眠模式 | 电量低 | 停止响应高功耗业务，尽量保护电池。 |

电源管理涉及以下业务：

- 可见光拉流
- 可见光手动拍照
- 可见光定时拍照
- 可见光录制短视频
- 可见光录像回放
- 红外拉流
- 红外手动拍照
- 红外录制短视频
- 红外定时拍照
- MIPI 摄像头拉流
- MIPI 摄像头拍照
- MIPI 摄像头录制短视频


## 2. 负载



| 负载 | 对应设备 | 说明 |
| --- | --- | --- |
| 云台可见光机芯负载 | 可见光机芯、云台控制能力 | 可见光业务的基础负载。 |
| 红外负载 | 红外机芯 | 红外拉流、拍照、短视频和回放需要使用。 |
| MIPI 负载 | MIPI 摄像头 | 使用独立 GPIO 上下电，不依赖云台或红外负载。 |


业务上的通道关系：

| 通道或设备类型 | 处理规则 |
| --- | --- |
| 通道一 | 通常按可见光通道处理。 |
| 通道二 | 按现场配置处理，可能是可见光或红外。 |
| MIPI 摄像头 | 按 MIPI 负载规则处理。 |

判断某个任务属于可见光、红外还是 MIPI，不只看通道号，还要看该通道绑定的设备类型。

## 3. 模式判定条件

电源模式由电池电压决定：

| 电压范围 | 目标模式 |
| --- | --- |
| 电压高于 12.99V | 全工作模式 |
| 电压大于 12.79V 且小于等于 12.99V | 基础模式 |
| 电压小于等于 12.79V | 休眠模式 |



为了避免电压抖动导致频繁切换，系统不会在电压第一次跨越阈值时立刻切换模式。目标模式必须持续稳定一段确认时间后，才认为模式切换有效。

当前确认时间为 30 分钟。也就是说，只有连续 30 分钟都满足同一个目标模式条件，系统才会确认目标模式有效。

## 4. 模式切换规则

### 4.1 正常切换

每次收到电量采样后，系统重新判断目标模式。

如果目标模式与当前模式一致，则清除候选模式，保持当前状态。

如果目标模式与当前模式不同，则进入候选状态，开始计时。候选状态持续到确认时间后，才正式切换。

### 4.2 忙状态延迟切换

如果确认需要切换模式，但设备正在执行业务，则当前模式值会先更新，进入模式后的上下电动作会延迟执行。

以下状态认为设备正在忙：

| 忙状态 | 是否阻止进入动作立即执行 |
| --- | --- |
| 正在打开设备 | 是 |
| 正在拉流 | 是 |
| 正在拍照 | 是 |
| 正在录制短视频 | 是 |
| 正在播放录像回放 | 是 |

单纯的已打开状态不认为是业务忙。因为已打开可能只是资源残留，如果把它当成忙状态，可能导致基础模式下一直无法下电。

当设备忙时，系统记录待执行的模式动作。等业务结束并检测到设备空闲后，再执行对应的进入动作，例如上电、下电、清空或恢复录像策略。

### 4.3 模式进入动作

| 目标模式 | 进入条件 | 进入动作 | 退出条件 |
| --- | --- | --- | --- |
| 全工作模式 | 电压持续高于 12.99V；确认时间达到要求；无阻塞模式切换的业务任务。 | 清除基础模式保活；打开系统接口供电；存在 MIPI 摄像头时开启 MIPI；工作时间段开启云台和红外；工作时间内恢复录像策略。 | 电压持续下降到基础模式或休眠模式区间；确认时间达到要求；若当前有业务任务，延迟到业务空闲后退出。 |
| 基础模式 | 电压持续处于 12.79V 到 12.99V 之间；确认时间达到要求；无阻塞模式切换的业务任务。 | 清除已有保活；保持基础接口供电；关闭云台和红外；关闭 MIPI 并释放 MIPI 摄像头连接；取消本地定时录像和短视频闹钟，并向可见光机芯下发空录像策略；重建定时拍照闹钟。 | 电压持续升高到全工作模式区间，或持续下降到休眠模式区间；若当前有业务任务，延迟到业务空闲后退出。 |
| 休眠模式 | 电压持续低于或等于 12.79V；确认时间达到要求；无阻塞模式切换的业务任务。 | 清除基础模式保活；关闭系统接口供电；关闭云台和红外；关闭 MIPI 并释放 MIPI 摄像头连接；取消本地定时录像和短视频闹钟，并向可见光机芯下发空录像策略；USB、RJ45 等高功耗能力进入低功耗状态。 | 电压恢复到基础模式或全工作模式区间；确认时间达到要求。 |

休眠模式下，高功耗业务一般不执行，平台下发拍照、拉流、短视频等任务时会直接返回或不响应实际设备动作。

## 5. 工作时间规则

系统支持配置工作时间段。

在全工作模式下，工作时间用于决定云台和红外是否常开：

| 场景 | 云台/红外策略 |
| --- | --- |
| 全工作模式工作时间内 | 云台和红外可保持开启，任务执行时不需要额外提前唤醒。 |
| 全工作模式非工作时间 | 云台和红外默认可关闭，任务执行前需要按需唤醒，任务结束后可再次关闭。 |

在基础模式和休眠模式下，系统整体按非工作状态处理：

| 模式 | 工作时间是否影响负载 | 处理策略 |
| --- | --- | --- |
| 基础模式 | 否 | 不因工作时间常开云台或红外，只在任务触发或定时拍照预热窗口内开启负载。 |
| 基础模式 MIPI | 否 | 不因工作时间常开，只在 MIPI 拍照、拉流、短视频等任务需要时临时上电。 |
| 休眠模式 | 否 | 不执行高功耗任务。 |

## 6. 基础模式保活机制

基础模式下，拍照、拉流、短视频、录像回放等业务入口会触发保活。

保活的目的：

- 避免刚完成一个任务后立即下电。
- 在短时间内连续执行任务时，不重复上下电。
- 给设备打开、登录、自检、拉流等动作保留稳定窗口。

当前普通业务保活时间为 15 分钟。

拍照和短视频完成后还有一层特殊处理：

| 业务完成场景 | 后续窗口 | 命中窗口时 | 未命中窗口时 |
| --- | --- | --- | --- |
| 可见光拍照或短视频完成 | 后续 10 分钟内存在通道一或通道二定时拍照任务 | 保持负载上电，并按 10 分钟重置保活。 | 清除当前保活并立即进入下电判断。 |
| 红外拍照或短视频完成 | 后续 15 分钟内存在通道一或通道二定时拍照任务 | 保持云台、红外等负载上电，并按 15 分钟重置保活。 | 清除当前保活并立即进入下电判断。 |
| MIPI 拍照、拉流或短视频结束 | 无 | 不参保活。 | 没有 MIPI 忙任务时立即关闭 MIPI 负载。 |

MIPI 摄像头不走可见光机芯的基础模式保活计时。基础模式下 MIPI 任务开始前按需上电，任务成功、失败或停止后调用 MIPI 空闲下电判断；如果没有 MIPI 摄像头正在打开、拉流、拍照或录制，则立即关闭 MIPI 负载。

保活期间的规则：

- 再次执行业务时，不应重复关闭再打开设备。
- 业务会刷新保活计时。
- 保活到期后，如果设备仍在忙，则继续延后保活。
- 保活到期后，如果设备空闲，才允许关闭云台和红外。

保活结束后再次执行任务：

- 如果云台或红外已经关闭，则需要重新唤醒。
- 重新唤醒后，设备必须重新等待就绪，不能复用旧的打开状态。

## 7. 定时拍照预唤醒规则

定时拍照除了正式拍照闹钟外，还需要根据通道类型生成提前唤醒闹钟。

| 通道类型 | 提前唤醒时间 | 提前开启负载 | 已进入预热窗口时 |
| --- | --- | --- | --- |
| 可见光定时拍照 | 拍照前 3 分钟 | 云台 | 立即触发一次预唤醒。 |
| 红外定时拍照 | 拍照前 10 分钟 | 云台、红外、必要 USB | 立即触发红外预唤醒。 |

### 7.1 可见光定时拍照

可见光定时拍照需要提前 3 分钟开启云台。

例如通道一定时拍照时间为 10:53:00，则系统应在 10:50:00 触发云台上电。

如果设置策略时已经进入拍照前 3 分钟窗口，则不再等待完整 3 分钟，而是立即触发一次预唤醒。

正式拍照时间到达后，执行拍照任务。

### 7.2 红外定时拍照

红外定时拍照需要提前 10 分钟开启云台和红外。

红外任务需要同时考虑：

- 云台上电
- 红外上电
- USB 上电
- 红外设备权限或初始化

如果设置策略时已经进入拍照前 10 分钟窗口，则立即触发红外预唤醒。

### 7.3 策略修改和删除

当某个通道的定时拍照策略被修改或删除时，该通道旧的正式拍照闹钟和提前唤醒闹钟都必须取消。

取消旧策略后，再根据新的策略表重新建立闹钟。

这样可以避免删除拍照策略后，提前开启云台或红外的闹钟仍然存在。

## 8. 全工作模式下业务执行逻辑

### 8.1 可见光拉流

工作时间内，云台通常已经处于开启状态。平台发起可见光拉流时，系统打开对应通道，等待设备就绪，然后开始推流。

非工作时间内，如果云台已关闭，拉流前需要先开启云台，再打开设备。

拉流结束后：

- 工作时间内，设备回到工作状态，云台可保持开启。
- 非工作时间内，如果没有其他任务，可关闭云台。

### 8.2 可见光手动拍照

平台发起手动拍照后，系统确认当前不是休眠模式，然后打开对应可见光设备。

工作时间内，通常直接执行拍照。

非工作时间内，如果云台已关闭，需要先开启云台，并等待设备就绪后再拍照。

拍照完成后：

- 工作时间内，恢复到录像预置位或保持工作状态。
- 非工作时间内，如果没有其他任务，可关闭云台。

### 8.3 可见光定时拍照

全工作模式工作时间内，定时拍照只注册正式拍照闹钟，不额外注册提前唤醒闹钟。

全工作模式非工作时间内，定时拍照需要提前 3 分钟开启云台。

拍照完成后，按当前是否工作时间决定是否关闭云台。

### 8.4 可见光录制短视频

平台发起短视频任务后，系统先判断该通道是否正在录制。

如果未录制，则打开设备，等待设备就绪，按策略执行预置位、巡航或巡检动作，然后开始录制短视频。

录制完成后：

- 工作时间内，恢复到录像预置位。
- 非工作时间内，如果没有其他任务，可关闭云台。

### 8.5 可见光录像回放

平台发起录像回放时，系统创建回放链路，打开设备，等待设备就绪，然后按指定时间段播放录像。

回放期间设备处于忙状态，不能执行会破坏回放的下电动作。

停止回放后：

- 关闭回放链路。
- 释放回放任务状态。
- 非工作时间内，如果没有其他任务，可关闭云台。

### 8.6 红外业务

全工作模式下，红外拉流、拍照、短视频和回放的流程与可见光类似，但红外任务需要额外保证红外负载和 USB 状态。

工作时间内，红外可保持开启。

非工作时间内，红外任务执行前需要上电红外和必要的 USB 能力，任务结束后如果没有其他任务，可关闭红外。

## 9. 基础模式下业务执行逻辑

基础模式是本设计的重点。该模式下默认云台和红外关闭，任务触发时才唤醒。

### 9.1 可见光拉流

进入条件：

- 当前模式为基础模式。
- 平台发起可见光拉流。
- 当前不是休眠模式。

执行流程：

- 标记基础模式业务活动。
- 开启云台。
- 启动保活计时。
- 打开可见光设备。
- 等待设备就绪。
- 设备就绪后开始拉流。

退出条件：

- 平台停止拉流。
- 拉流失败。
- 网络链路断开或设备打开失败。

退出动作：

- 停止拉流。
- 释放任务状态。
- 进入保活期。
- 保活期内不关闭云台。
- 保活期结束且没有其他任务、没有拍照预热窗口时，关闭云台和红外。

保活期间再次拉流：

- 直接复用已唤醒的负载状态。
- 刷新保活计时。
- 不应出现先下电再上电的动作。

保活结束后再次拉流：

- 重新开启云台。
- 重新等待设备就绪。

### 9.2 可见光手动拍照

进入条件：

- 当前模式为基础模式。
- 平台发起可见光拍照。

执行流程：

- 标记基础模式业务活动。
- 开启云台。
- 启动保活计时。
- 打开可见光设备。
- 等待设备就绪。
- 执行拍照。

退出动作：

- 拍照成功或失败后释放任务状态。
- 如果后续 10 分钟内存在通道一或通道二定时拍照任务，则保持云台上电并按 10 分钟重置保活。
- 如果后续 10 分钟内没有通道一或通道二定时拍照任务，则立即进入下电判断。

保活期间再次拍照：

- 不重新下电。
- 直接执行设备打开或拍照流程。
- 刷新保活时间。

保活结束后再次拍照：

- 重新唤醒云台。
- 重新等待设备就绪。

### 9.3 可见光定时拍照

进入条件：

- 当前模式为基础模式。
- 存在可见光定时拍照策略。

执行流程：

- 设置策略时生成正式拍照闹钟。
- 同时生成拍照前 3 分钟的云台预唤醒闹钟。
- 预唤醒时间到达后开启云台。
- 正式拍照时间到达后执行拍照。

退出动作：

- 拍照完成后按可见光拍照完成规则处理。
- 如果后续 10 分钟内存在通道一或通道二定时拍照任务，则保持云台上电并按 10 分钟重置保活。
- 如果后续 10 分钟内没有通道一或通道二定时拍照任务，则立即进入下电判断。

特殊规则：

- 如果删除或修改拍照策略，正式拍照闹钟和提前唤醒闹钟都要同步取消或重建。
- 如果当前已经在拍照前 3 分钟窗口内，预唤醒应立即执行。

### 9.4 可见光录制短视频

进入条件：

- 当前模式为基础模式。
- 平台发起短视频录制。
- 当前通道没有正在录制的任务。

执行流程：

- 将短视频任务加入队列。
- 如果当前没有短视频正在执行，则取出队列中的任务。
- 标记基础模式业务活动。
- 开启云台。
- 启动保活计时。
- 打开可见光设备。
- 等待设备就绪。
- 按任务参数执行预置位、巡航或巡检。
- 开始录制短视频。

退出动作：

- 录制完成或失败后释放任务状态。
- 继续执行队列中的下一个短视频任务。
- 如果队列为空，则按可见光短视频完成规则处理。
- 如果后续 10 分钟内存在通道一或通道二定时拍照任务，则保持云台上电并按 10 分钟重置保活。
- 如果后续 10 分钟内没有通道一或通道二定时拍照任务，则立即进入下电判断。

保活期间再次录制：

- 新任务进入队列。
- 不重复下电。
- 执行时刷新保活。

### 9.5 可见光录像回放

进入条件：

- 当前模式为基础模式。
- 平台请求录像文件列表、录像文件数量、录像回放或回放控制。

执行流程：

- 查询文件数量或列表时，先标记基础模式活动并启动保活。
- 在基础模式下，首次查询可优先返回缓存的录像文件信息，用于避免还未完全唤醒时查询失败。
- 真正播放回放时，开启云台，打开设备，建立回放链路。
- 等待设备就绪后开始播放录像。

退出动作：

- 停止回放时关闭回放链路。
- 释放回放任务状态。
- 进入保活期。
- 保活结束后，如果没有其他任务和拍照预热窗口，则关闭云台。

保活期间再次回放：

- 不重复下电。
- 刷新保活。
- 可继续查询或播放。

### 9.6 红外拉流

进入条件：

- 当前模式为基础模式。
- 平台发起红外通道拉流。

执行流程：

- 标记基础模式业务活动。
- 开启云台、红外和必要的 USB。
- 启动保活计时。
- 如果红外设备需要 USB 权限或设备注册，则完成相关准备。
- 打开红外设备。
- 等待设备就绪。
- 设备就绪后开始拉流。

退出动作：

- 停止拉流或拉流失败后释放任务状态。
- 进入保活期。
- 保活结束后，如果没有其他任务，且不处于红外拍照前 10 分钟窗口，则关闭红外。
- 如果关闭红外时需要同时关闭云台，也必须确认云台没有被其他可见光任务占用。

### 9.7 红外手动拍照

进入条件：

- 当前模式为基础模式。
- 平台发起红外拍照。

执行流程：

- 标记基础模式业务活动。
- 开启云台、红外和必要的 USB。
- 启动保活计时。
- 打开红外设备。
- 等待设备就绪。
- 执行红外拍照。
- 如果启用红外拼接或双光融合，则继续执行对应的多图或可见光辅助拍照流程。

退出动作：

- 拍照完成后按红外拍照完成规则处理。
- 如果后续 15 分钟内存在通道一或通道二定时拍照任务，则保持云台、红外等负载上电并按 15 分钟重置保活。
- 如果后续 15 分钟内没有通道一或通道二定时拍照任务，则立即进入下电判断。

### 9.8 红外定时拍照

进入条件：

- 当前模式为基础模式。
- 存在红外定时拍照策略。

执行流程：

- 设置策略时生成正式拍照闹钟。
- 同时生成拍照前 10 分钟的预唤醒闹钟。
- 预唤醒时开启云台、红外和必要 USB。
- 正式拍照时间到达后执行红外拍照。

退出动作：

- 拍照完成后按红外拍照完成规则处理。
- 如果后续 15 分钟内存在通道一或通道二定时拍照任务，则保持云台、红外等负载上电并按 15 分钟重置保活。
- 如果后续 15 分钟内没有通道一或通道二定时拍照任务，则立即进入下电判断。

### 9.9 红外录制短视频

进入条件：

- 当前模式为基础模式。
- 平台发起红外短视频录制。

执行流程：

- 标记基础模式业务活动。
- 开启云台、红外和必要 USB。
- 启动保活计时。
- 打开红外设备。
- 等待设备就绪。
- 按任务参数执行预置位、巡航或巡检。
- 开始录制短视频。

退出动作：

- 录制完成或失败后释放任务状态。
- 按红外短视频完成规则处理。
- 如果后续 15 分钟内存在通道一或通道二定时拍照任务，则保持云台、红外等负载上电并按 15 分钟重置保活。
- 如果后续 15 分钟内没有通道一或通道二定时拍照任务，则立即进入下电判断。

### 9.10 红外录像回放

进入条件：

- 当前模式为基础模式。
- 平台发起红外录像回放。

执行流程：

- 标记基础模式业务活动。
- 开启云台、红外和必要 USB。
- 启动保活计时。
- 打开红外设备。
- 建立回放链路。
- 等待设备就绪后开始播放录像。

退出动作：

- 停止回放时关闭回放链路。
- 释放回放任务状态。
- 进入保活期。
- 保活结束后，如果没有其他任务和红外拍照预热窗口，则关闭红外。

### 9.11 MIPI 摄像头业务

进入条件：

- 当前模式为基础模式。
- 平台发起 MIPI 摄像头拉流、拍照或短视频录制。
- 当前不是休眠模式。

执行流程：

- 如果另一个摄像头正在直播，会先停止另一路直播。
- 如果 MIPI 摄像头正在录制，则拉流会优先让录制继续，拍照前会停止当前录制。
- 任务开始前开启 MIPI 负载。
- 打开 MIPI 摄像头设备。
- 等待设备就绪。
- 设备就绪后执行拉流、拍照或短视频录制。

退出动作：

- 拉流停止、拍照成功或失败、短视频录制成功或失败、设备打开失败、推流连接失败时，都会进入 MIPI 空闲下电判断。
- 如果仍有 MIPI 摄像头正在打开、拉流、拍照或录制，则暂不下电。
- 如果没有 MIPI 摄像头任务正在执行，则关闭 MIPI 负载，并释放 MIPI 摄像头连接状态。

特殊规则：

- MIPI 摄像头不参与 DVR 基础模式 15 分钟保活。
- MIPI 摄像头没有拍照前 3 分钟或红外拍照前 10 分钟预热闹钟。
- 全工作模式下 MIPI 负载默认开启；基础模式和休眠模式下 MIPI 负载默认关闭。

## 10. 休眠模式下业务执行逻辑

休眠模式用于保护电池。

进入休眠模式后：

| 对象 | 休眠模式进入动作 |
| --- | --- |
| 云台 | 关闭。 |
| 红外 | 关闭。 |
| MIPI 负载 | 关闭。 |
| 高功耗接口 | 关闭。 |
| 基础模式保活状态 | 清空。 |

业务处理规则：

| 业务 | 休眠模式处理 |
| --- | --- |
| 手动拍照 | 不执行。 |
| 拉流 | 不执行。 |
| 短视频录制 | 不执行。 |
| MIPI 摄像头拍照、拉流、短视频录制 | 不执行。 |
| 录像回放 | 不执行。 |
| 定时拍照 | 不执行。 |

休眠模式退出后，根据新的目标模式重新恢复业务能力。

## 11. 下电保护条件

系统不是所有空闲时刻都可以立即下电。以下情况会阻止下电：

| 阻止下电的情况 | 保护对象 |
| --- | --- |
| 设备正在打开 | 对应业务负载 |
| 设备正在拉流 | 对应业务负载 |
| 设备正在拍照 | 对应业务负载 |
| 设备正在录制短视频 | 对应业务负载 |
| 设备正在播放录像回放 | 对应业务负载 |
| MIPI 摄像头正在打开、拉流、拍照或录制 | MIPI 负载 |
| 当前处于基础模式保活期 | 云台、红外 |
| 可见光拍照任务将在 3 分钟内发生 | 云台 |
| 红外拍照任务将在 10 分钟内发生 | 云台、红外 |

只有同时满足以下条件，基础模式才允许关闭云台和红外：

| 条件 | 要求 |
| --- | --- |
| 业务状态 | 无正在执行的业务。 |
| 保活状态 | 不处于保活期。 |
| 可见光预热窗口 | 不处于可见光拍照前 3 分钟窗口。 |
| 红外预热窗口 | 不处于红外拍照前 10 分钟窗口。 |

MIPI 负载的下电条件独立判断：

| 条件 | 要求 |
| --- | --- |
| 当前模式 | 处于基础模式或休眠模式。 |
| MIPI 忙状态 | 没有 MIPI 摄像头正在打开、拉流、拍照或录制。 |
| 出口状态 | MIPI 直播停止、拍照完成或失败、短视频录制完成或失败、设备打开失败、推流连接失败等出口已经释放对应任务状态。 |

## 12. 设备重新就绪规则

当云台或红外真正下电前，需要清理设备运行状态。

这样做的目的是避免下一次上电后复用旧状态，导致设备没有重新等待就绪。

正确行为是：

- 负载下电前，释放旧的设备连接状态。
- 下一次任务触发时，重新上电。
- 重新打开设备。
- 重新等待设备就绪。
- 设备就绪后再执行拍照、拉流、短视频或回放。

MIPI 负载下电前会关闭 MIPI 摄像头连接。下一次基础模式 MIPI 任务触发时，需要重新开启 MIPI GPIO、重新打开摄像头，并重新等待设备就绪。

该规则对基础模式尤其重要。因为基础模式下设备频繁上下电，如果复用旧状态，可能出现还未真正连上设备就开始登录、拍照或拉流的情况。

## 13. 异常和失败处理

### 13.1 设备打开失败

如果任务打开设备失败：

- 当前任务失败。
- 释放任务状态。
- 必要时尝试重启对应负载。
- 进入保活或下电判断流程。

### 13.2 拉流失败

如果拉流失败：

- 停止当前拉流任务。
- 关闭或重建网络链路。
- 对应负载可根据保活规则决定是否保持。

### 13.3 拍照失败

如果拍照失败：

- 上报拍照失败。
- 释放拍照任务状态。
- 可按补拍逻辑再次尝试。
- 补拍仍需遵守当前电源模式规则。

### 13.4 回放失败

如果回放失败：

- 停止回放任务。
- 关闭回放链路。
- 释放任务状态。
- 按保活规则决定是否保持负载。
